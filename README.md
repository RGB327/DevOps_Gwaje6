# DHCP
> Dynamic Host Configuration Protocol의 약자로, 네트워크 기기(컴퓨터, 스마트폰 등)에 IP 주소, 서브넷 마스크, 기본 게이트웨이, DNS 서버 주소 등 필수 네트워크 설정 정보를 자동으로 할당하는 프로토콜이다.

**사용처**
- **공유기**: 기기가 와이파이에 연결될 때 공유기가 IP주소를 자동 할당함
- **인스턴스 생성**: 클라우드 서비스에서 새로운 서버 인스턴스를 생성할 때 통신이 가능하도록 네트워크 정보를 할당

**작동 방식**
4단계로 이루어져있으며 DORA라고 부름
1. **Discover**: 클라이언트가 브로드캐스트로 DHCP 서버를 찾음
2. **Offer**: 서버가 클라이언트에게 사용 가능한 IP를 제안
3. **Request**: 클라이언트가 제안받은 IP를 사용하겠다고 요청
4. **Acknowledgement**: 서버가 최종 승인

*이 떄 DHCP가 빌려주는 주소는 임대 기간 동안만 사용 가능하고 이는 연장이 가능하다*
# 서브넷 마스크

> IP주소에서 네트워크 영역과 호스트 영역을 구분하여 로컬 네트워크 범위를 정의하는 32비트 수


**구조**: IP주소와 같이 2진수 32비트로 표현되며 1은 네트워크 부분, 0은 호스트 부분이다

**규칙**: 1은 제일 왼쪽부터 연속적으로 나와야하며 중간에 0이 나왔다 1이 나올 수는 없다.
(11111111 11111111 11111111 00000000) -> O
(11111111 11110111 10101010 00001111) -> X

**활용**: 서브넷 마스크와 IP 주소를 AND 연산하여 네트워크 주소를 알아낼 수 있음

*호스트 부분이 모두 0인 주소와 1인 주소는 각각 네트워크 주소,브로드캐스트 주소로 사용된다.* 

### 클래스풀에서의 주소 구분
> Classful 주소 구조는 IP주소체계 초기에 사용하던 방식으로 첫 옥텟 값에 따라 네트워크 크기를 클래스로 나누어 관리하던 체계이다


#### 일반적으로 사용하는 클래스들의 범위
*0과 127은 특수 목적으로 사용됨(DHCP요청, 루프백 등)*
- **A클래스** 
    - 0~126(첫 옥텟의 앞부분 0) , 서브넷 마스크 255.0.0.0
    - 대형 기관이나 국가 단위에서 사용하며 최대 가용 호스트 수는 16,777,214개
- **B클래스**
    - 128~191(첫 옥텟 앞부분 10) , 서브넷 마스크 255.255.0.0
    - 학교나 중대형 기업에서 사용하며 최대 가용 호스트 수는 65,534개
- **C클래스**
    - 192~223(첫 옥텟 앞부분 110) , 서브넷 마스크 255.255.255.0
    - 일반적인 가정이나 소규모 사무실에서 사용하며 최대 가용 호스트 수는 254개

## CIDR
> Classless Inter-Domain Routing의 약자로 단어 그대로 클래스 없는 도메인 간 라우팅 기법이다.

**만들어진 이유**
클래스풀 주소체계에서 애매한 수의 주소가 필요할 때 IP낭비가 심함, 이를 해결하기 위해 클래스가 없는 주소 체제를 만듬

**장점**
- **IP주소의 효율적 활용**
    - 필요한 호스트 수 만큼 서브넷 마스크를 쪼개서 할당 가능
- **라우팅 테이블 크기 감소**
    - 슈퍼네팅을 통해 여러 연속된 소규모 IP를 하나로 묶어 표현해 관리해야 할 경로가 줄어 라우팅 테이블의 크기가 감소함



## CIDR 표기법
> IP주소 뒤에 /수 형식으로 서브넷 마스크를 표기하는 방식, Prefix라고도 부른다

이 때 수 부분에는 서브넷 마스크의 1의 개수가 들어간다
예시: 255.255.255.0(11111111.11111111.11111111.00000000) -> /24


## 서브넷팅

문제1. 
**네트워크 주소** = 192.168.1.0
**브로드캐스트 주소** = 192.168.1.255
**사용 가능한 호스트 수**: 254개

문제2.
**서브넷 마스크** = 255.255.255.192 
**사용 가능한 호스트 수** = 62개
**첫 번째 호스트 ~ 마지막 호스트 범위** = 192.168.1.1~192.168.1.62

문제3. 
**빌려야하는 비트 수** = 2개
**각 서브넷의 네트워크 주소**: 172.16.0.0 , 172.16.64.0 , 172.16.128.0 , 172.16.192.0 
**각 서브넷당 호스트 수**: 2의 14승 = 16384-2 = 16382


## 서브넷팅 계산법
**가용 호스트 개수 구하는 공식**
2의 (32-prefix)승-2

**-2를 해주는 이유**: 첫번쨰 주소와 마지막 주소는 네트워크 주소와 브로드캐스트 주소로 사용되기 때문

서브넷의 개수는 2의 (빌려온 호스트 비트 수)제곱만큼 증가한다
